CSS Grid Layout

CSS Grid Layout is a two-dimensional layout system designed specifically for web user interfaces. Grid allows you to create complex layouts with rows and columns, making it easier to design web pages without having to rely on floats and positioning.

Creating a Grid Container

To create a grid container, you need to set the display property to grid or inline-grid:

.container {
    display: grid; /* or inline-grid */
}

Grid Container Properties

These properties are applied to the grid container:

Property Description
grid-template-columns Defines the columns of the grid with a space-separated list of values
grid-template-rows Defines the rows of the grid with a space-separated list of values
grid-template-areas Defines named grid areas
grid-column-gap Sets the gap between columns
grid-row-gap Sets the gap between rows
grid-gap Shorthand for grid-row-gap and grid-column-gap
justify-items Aligns grid items along the row axis
align-items Aligns grid items along the column axis
justify-content Aligns the grid within the container along the row axis
align-content Aligns the grid within the container along the column axis

Grid Item Properties

These properties are applied to the grid items (children of the grid container):

Property Description
grid-column-start Specifies the start position of an item on the column axis
grid-column-end Specifies the end position of an item on the column axis
grid-row-start Specifies the start position of an item on the row axis
grid-row-end Specifies the end position of an item on the row axis
grid-column Shorthand for grid-column-start and grid-column-end
grid-row Shorthand for grid-row-start and grid-row-end
grid-area Specifies a grid item's location in a named grid area, or shorthand for grid-row-start, grid-column-start, grid-row-end, grid-column-end
justify-self Aligns an individual grid item in the inline direction
align-self Aligns an individual grid item in the block direction

The fr Unit

The fr unit represents a fraction of the available space in the grid container. This unit makes it easy to create flexible grid layouts:

.container {
    display: grid;
    grid-template-columns: 1fr 2fr 1fr; /* First column takes 1/4, second 2/4, third 1/4 of available space */
    grid-gap: 20px;
}

Common Grid Layouts

Basic Grid

.container {
    display: grid;
    grid-template-columns: repeat(3, 1fr); /* Creates 3 equal columns */
    grid-gap: 20px;
}

Holy Grail Layout

.container {
    display: grid;
    grid-template-areas:
        "header header header"
        "nav content sidebar"
        "footer footer footer";
    grid-template-columns: 200px 1fr 200px;
    grid-template-rows: auto 1fr auto;
    min-height: 100vh;
}

.header { grid-area: header; }
.nav { grid-area: nav; }
.content { grid-area: content; }
.sidebar { grid-area: sidebar; }
.footer { grid-area: footer; }

Grid with Auto-Fit

.container {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
    grid-gap: 20px;
}

Grid vs. Flexbox

While both CSS Grid and Flexbox are powerful layout tools, they serve different purposes:

  • Dimensions: Grid is two-dimensional (rows and columns), while Flexbox is one-dimensional (row or column)
  • Layout approach: Grid is layout-first (define the grid, then place items), while Flexbox is content-first (items define their own layout)
  • Alignment: Grid excels at aligning items in both dimensions, while Flexbox focuses on one dimension
  • Use cases: Grid is ideal for overall page layout, while Flexbox works best for components and small-scale layouts

Browser Support

CSS Grid is supported in all modern browsers, including:

  • Chrome 57+
  • Firefox 52+
  • Safari 10.1+
  • Edge 16+
  • Opera 44+
  • iOS Safari 10.3+
  • Android Browser 81+
