HTML Tables - Organizing Data in Rows and Columns

HTML tables are used to display data in a structured grid format consisting of rows and columns. They're perfect for presenting tabular data like schedules, product comparisons, financial statements, and other information that benefits from a clear, organized layout.

Basic Table Structure

A basic HTML table is created using these elements:

  • <table> - Container for the entire table
  • <tr> - Table row
  • <th> - Table header cell
  • <td> - Table data cell

These elements work together to form the structure of your table with headers and data organized in rows and columns.

Table Headers and Data

A typical table includes headers and data cells:

<table>
  <tr>
    <th>Name</th>
    <th>Age</th>
    <th>Country</th>
  </tr>
  <tr>
    <td>John Smith</td>
    <td>28</td>
    <td>USA</td>
  </tr>
</table>

Headers (<th>) typically appear bold and centered by default, while data cells (<td>) are regular text aligned to the left.

Table Structure Elements

For better organization, tables can include:

  • <thead> - Groups header content
  • <tbody> - Groups body content
  • <tfoot> - Groups footer content
  • <caption> - Provides a title/description

These elements improve accessibility and make styling more manageable, especially for larger tables.

Table Attributes and Styling

HTML tables can be customized with various attributes and CSS to improve appearance:

<table border="1" cellpadding="10" cellspacing="0" style="width: 80%;">
  <caption>Monthly Sales Report</caption>
  <tr>
    <th>Product</th>
    <th>January</th>
    <th>February</th>
    <th>March</th>
  </tr>
  <tr>
    <td>Widgets</td>
    <td>250</td>
    <td>315</td>
    <td>290</td>
  </tr>
</table>

Note: While HTML attributes like border, cellpadding, and cellspacing are shown above, modern best practice is to use CSS for styling tables.

Table CSS Styling Example

table {
  width: 80%;
  border-collapse: collapse;
  margin: 20px auto;
}

th, td {
  border: 1px solid #ddd;
  padding: 8px;
  text-align: left;
}

th {
  background-color: #f2f2f2;
  font-weight: bold;
}

tr:nth-child(even) {
  background-color: #f9f9f9;
}

caption {
  caption-side: top;
  font-weight: bold;
  padding: 10px;
}

Table Spanning Columns and Rows

Tables can have cells that span multiple columns or rows using the colspan and rowspan attributes:

Attribute Description Example
colspan Specifies how many columns a cell should span <td colspan="2">Spans 2 columns</td>
rowspan Specifies how many rows a cell should span <td rowspan="3">Spans 3 rows</td>

Spanning Example

<table border="1">
  <tr>
    <th>Name</th>
    <th colspan="2">Contact Info</th>
  </tr>
  <tr>
    <td>John Smith</td>
    <td>Phone: 555-1234</td>
    <td>Email: john@example.com</td>
  </tr>
  <tr>
    <td>Jane Doe</td>
    <td>Phone: 555-5678</td>
    <td>Email: jane@example.com</td>
  </tr>
</table>

Complex Table Structure

For more complex tables, use table sections and additional attributes:

<table>
  <caption>Quarterly Sales Report</caption>
  <thead>
    <tr>
      <th rowspan="2">Department</th>
      <th colspan="4">2025 Quarters</th>
    </tr>
    <tr>
      <th>Q1</th>
      <th>Q2</th>
      <th>Q3</th>
      <th>Q4</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Electronics</td>
      <td>$10,000</td>
      <td>$12,500</td>
      <td>$15,000</td>
      <td>$18,000</td>
    </tr>
    <tr>
      <td>Clothing</td>
      <td>$8,000</td>
      <td>$9,500</td>
      <td>$11,000</td>
      <td>$15,500</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th>Total</th>
      <td>$18,000</td>
      <td>$22,000</td>
      <td>$26,000</td>
      <td>$33,500</td>
    </tr>
  </tfoot>
</table>

Table Accessibility

Make your tables accessible to all users by following these best practices:

  • Use proper table headers (<th>) for columns and rows
  • Add a descriptive <caption> to explain the table's purpose
  • Use scope attributes on headers: scope="col" for column headers, scope="row" for row headers
  • Use id and headers attributes for complex tables to associate data cells with headers
  • Avoid using tables for layout purposes (use CSS for page layout instead)
  • Keep tables simple or break complex data into multiple tables
  • Add a summary attribute or aria-describedby for screen readers
  • Ensure adequate color contrast for all table elements

Tables vs. CSS Grid/Flexbox

Use Tables For Use CSS Grid/Flexbox For
Tabular data display Page layouts
Data comparison Navigation menus
Calendars Card layouts
Price lists Image galleries
Financial reports Form layouts

Responsive Tables

Tables can be challenging on smaller screens. Here are some techniques for making responsive tables:

/* Method 1: Horizontal scrolling */
.table-container {
  width: 100%;
  overflow-x: auto;
}

/* Method 2: Collapse table on small screens */
@media screen and (max-width: 600px) {
  table, thead, tbody, th, td, tr {
    display: block;
  }
  
  thead tr {
    position: absolute;
    top: -9999px;
    left: -9999px;
  }
  
  td {
    position: relative;
    padding-left: 50%;
  }
  
  td:before {
    position: absolute;
    left: 6px;
    width: 45%;
    padding-right: 10px;
    white-space: nowrap;
    content: attr(data-label);
    font-weight: bold;
  }
}
