HTML Attributes - Adding Information to Elements

HTML attributes provide additional information about HTML elements. They are always specified in the start tag and usually come in name/value pairs like: name="value". Let's explore how attributes enhance HTML elements.

Attribute Syntax

HTML attributes have a simple syntax:

<tagname attribute="value">Content</tagname>

Key points about attributes:

  • Specified in the start tag
  • Come in name/value pairs
  • Values are enclosed in quotes
  • Some attributes can be used without values

Common Attributes

Some attributes can be used with most HTML elements:

  • id - Unique identifier
  • class - Group identifier
  • style - Inline CSS styling
  • title - Additional information
  • lang - Language specification

These global attributes can be used on any HTML element.

Element-Specific Attributes

Some attributes are specific to certain elements:

  • href - for <a> (links)
  • src - for <img> (images)
  • alt - for <img> (alternative text)
  • width/height - for various elements
  • type - for <input> elements

These attributes provide essential functionality for their respective elements.

Examples of HTML Attributes

Example Description
<a href="https://example.com">Link</a> The href attribute specifies the URL of the page the link goes to
<img src="image.jpg" alt="Description"> src specifies the image path, alt provides alternative text
<p style="color:blue;">Blue text</p> style attribute adds inline CSS styling
<div class="container">Content</div> class attribute groups elements for CSS styling
<button disabled>Click me</button> disabled is a boolean attribute with no value

Boolean Attributes

Some attributes don't need a value. These are called boolean attributes and their presence indicates "true":

<input type="text" disabled>
<button disabled>Click me</button>
<input type="checkbox" checked>

Tips for Using Attributes

  • Always use lowercase attribute names
  • Always quote attribute values
  • Never leave spaces around the equals sign
  • For boolean attributes, just the attribute name is enough
  • Use data-* attributes for custom data
