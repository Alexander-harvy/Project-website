PHP Functions

Functions are blocks of code that can be reused throughout your application. PHP has over 1,000 built-in functions, and you can also create your own custom functions to perform specific tasks.

Creating a PHP Function

In PHP, a function is created with the function keyword, followed by the function name and parentheses. Function names follow the same rules as variable names in PHP.


<?php
// Syntax
function functionName() {
    // code to be executed
}

// Example of a simple function
function sayHello() {
    echo "Hello, World!";
}

// Calling the function
sayHello(); // Outputs: Hello, World!
?>

Function Parameters

Parameters allow you to pass information to a function. You can specify as many parameters as you want, separated by commas.


<?php
// Function with parameters
function greet($name) {
    echo "Hello, $name!";
}

// Calling the function with an argument
greet("John"); // Outputs: Hello, John!

// Function with multiple parameters
function greetWithTime($name, $time) {
    echo "Good $time, $name!";
}

greetWithTime("John", "morning"); // Outputs: Good morning, John!
?>

Default Parameter Values

PHP allows you to define default values for function parameters. If a function is called without passing a value for a parameter with a default value, the default value will be used.


<?php
// Function with default parameter value
function setHeight($height = 50) {
    echo "The height is: $height";
}

setHeight(80); // Outputs: The height is: 80
setHeight();   // Outputs: The height is: 50 (default value is used)
?>

Returning Values

Functions can return values using the return statement. Once a return statement is executed, the function stops running.


<?php
// Function that returns a value
function sum($a, $b) {
    return $a + $b;
}

// Store the returned value in a variable
$result = sum(5, 10);
echo "The sum is: $result"; // Outputs: The sum is: 15

// Use the returned value directly
echo "The sum is: " . sum(5, 10); // Outputs: The sum is: 15

// Function with early return
function checkNumber($num) {
    if ($num <= 0) {
        return "Number must be positive";
    }
    return "Number is positive: $num";
}

echo checkNumber(-5); // Outputs: Number must be positive
echo checkNumber(10); // Outputs: Number is positive: 10
?>

Function Scope

Variables declared inside a function are local to that function and can only be accessed within it. To access global variables inside a function, you need to use the global keyword.


<?php
// Global variable
$x = 10;

function localScope() {
    // Local variable
    $y = 5;
    echo "Local variable y: $y
"; // Accessible
    
    // Trying to access global variable (won't work)
    echo "Trying to access x: $x
"; // Will output an empty value or warning
}

localScope();

// Using the global keyword
function globalScope() {
    global $x;
    echo "Global variable x: $x
"; // Now accessible: 10
}

globalScope();

// Using the $GLOBALS array
function globalsArray() {
    echo "Global variable x using \$GLOBALS: " . $GLOBALS['x'] . "
"; // Accessible: 10
}

globalsArray();
?>

Static Variables

Static variables retain their value between function calls. They are declared using the static keyword.


<?php
function counter() {
    static $count = 0;
    $count++;
    echo "Function called $count time(s)
";
}

counter(); // Outputs: Function called 1 time(s)
counter(); // Outputs: Function called 2 time(s)
counter(); // Outputs: Function called 3 time(s)
?>

Passing Arguments by Reference

By default, function arguments are passed by value (which means a copy of the value is passed). To pass by reference (which means passing the variable itself), use an ampersand (&) before the parameter name.


<?php
// Passing by value (default)
function addFive($num) {
    $num += 5;
}

$origNum = 10;
addFive($origNum);
echo "Original number: $origNum
"; // Still 10, not affected

// Passing by reference
function addTen(&$num) {
    $num += 10;
}

$origNum = 10;
addTen($origNum);
echo "Original number after reference function: $origNum
"; // Now 20, has been modified
?>

Anonymous Functions (Closures)

PHP supports anonymous functions (also known as closures), which are functions without names that can be assigned to variables or passed as arguments to other functions.


<?php
// Anonymous function assigned to a variable
$greet = function($name) {
    echo "Hello, $name!";
};

// Calling the anonymous function
$greet("John"); // Outputs: Hello, John!

// Anonymous function as an argument to another function
$numbers = [1, 2, 3, 4, 5];
$squaredNumbers = array_map(function($n) {
    return $n * $n;
}, $numbers);

print_r($squaredNumbers);
// Outputs: Array ( [0] => 1 [1] => 4 [2] => 9 [3] => 16 [4] => 25 )
?>

Arrow Functions (PHP 7.4+)

PHP 7.4 introduced arrow functions, which provide a shorter syntax for simple functions. They are especially useful for short callbacks.


<?php
// Traditional anonymous function
$numbers = [1, 2, 3, 4, 5];
$squaredTraditional = array_map(function($n) {
    return $n * $n;
}, $numbers);

// Arrow function equivalent (PHP 7.4+)
$squaredArrow = array_map(fn($n) => $n * $n, $numbers);

print_r($squaredArrow);
// Outputs: Array ( [0] => 1 [1] => 4 [2] => 9 [3] => 16 [4] => 25 )
?>

Recursive Functions

A recursive function is a function that calls itself. Recursive functions can be useful for tasks like calculating factorials or traversing tree-like structures.


<?php
// Recursive function to calculate factorial
function factorial($n) {
    if ($n <= 1) {
        return 1;
    } else {
        return $n * factorial($n - 1);
    }
}

echo "Factorial of 5: " . factorial(5) . "
"; // Outputs: 120
// How it works: 5 * factorial(4) = 5 * 24 = 120
// factorial(4) = 4 * factorial(3) = 4 * 6 = 24
// factorial(3) = 3 * factorial(2) = 3 * 2 = 6
// factorial(2) = 2 * factorial(1) = 2 * 1 = 2
// factorial(1) = 1
?>

Built-in PHP Functions

PHP has many built-in functions for various tasks. Here are some commonly used categories:

String Functions


<?php
$str = "Hello, World!";

// Getting the length of a string
echo strlen($str) . "
"; // Outputs: 13

// Counting words in a string
echo str_word_count($str) . "
"; // Outputs: 2

// Reversing a string
echo strrev($str) . "
"; // Outputs: !dlroW ,olleH

// Finding text within a string
echo strpos($str, "World") . "
"; // Outputs: 7 (position of "World")

// Replacing text within a string
echo str_replace("World", "PHP", $str) . "
"; // Outputs: Hello, PHP!
?>

Array Functions


<?php
$fruits = ["apple", "banana", "orange"];

// Count elements in an array
echo count($fruits) . "
"; // Outputs: 3

// Adding elements to an array
array_push($fruits, "mango");
print_r($fruits); // Adds "mango" to the end

// Removing the last element
$last = array_pop($fruits);
echo "Removed: $last
"; // Outputs: Removed: mango

// Sorting arrays
sort($fruits);
print_r($fruits); // Sorts alphabetically
?>

Date and Time Functions


<?php
// Current timestamp
echo time() . "
"; // Unix timestamp (seconds since Jan 1 1970)

// Formatting a date
echo date("Y-m-d H:i:s") . "
"; // Outputs: 2025-04-10 10:15:30 (example)

// Get day of the week
echo date("l") . "
"; // Outputs: Thursday (for current date of Thursday, April 10, 2025)

// Create a date from a string
$date = strtotime("next Sunday");
echo date("Y-m-d", $date) . "
"; // Outputs: 2025-04-13 (next Sunday from April 10, 2025)
?>

Real-World Examples

Example 1: Form Validation Function


<?php
function validateEmail($email) {
    // Remove whitespace
    $email = trim($email);
    
    // Check if it's empty
    if (empty($email)) {
        return "Email is required";
    }
    
    // Validate email format
    if (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
        return "Invalid email format";
    }
    
    return true; // Email is valid
}

// Usage
$email = "user@example.com";
$validation = validateEmail($email);

if ($validation === true) {
    echo "Email is valid";
} else {
    echo "Error: " . $validation;
}
?>

Example 2: Database Connection Function


<?php
function connectToDatabase() {
    $host = "localhost";
    $user = "username";
    $pass = "password";
    $dbname = "mydb";
    
    // Create connection
    $conn = new mysqli($host, $user, $pass, $dbname);
    
    // Check connection
    if ($conn->connect_error) {
        die("Connection failed: " . $conn->connect_error);
    }
    
    return $conn;
}

// Usage
$conn = connectToDatabase();
echo "Connected successfully";

// Don't forget to close the connection when done
$conn->close();
?>

Example 3: Utility Functions for a Blog


<?php
// Format dates in a user-friendly way
function formatDate($dateString) {
    $date = new DateTime($dateString);
    return $date->format('F j, Y'); // e.g., April 10, 2025
}

// Create URL-friendly slugs from titles
function createSlug($title) {
    // Remove special characters
    $slug = preg_replace('/[^A-Za-z0-9-]+/', '-', $title);
    // Convert to lowercase
    $slug = strtolower($slug);
    // Remove hyphens from beginning and end
    $slug = trim($slug, '-');
    return $slug;
}

// Truncate text to a specific length with ellipsis
function truncateText($text, $length = 100) {
    if (strlen($text) <= $length) {
        return $text;
    }
    
    // Truncate to the specified length
    $text = substr($text, 0, $length);
    // Make sure we don't cut off in the middle of a word
    $text = substr($text, 0, strrpos($text, ' '));
    return $text . '...';
}

// Example usage
$postTitle = "How to Use PHP Functions Effectively in Your Projects";
$postDate = "2025-04-10";
$postContent = "PHP functions are an essential part of creating maintainable and reusable code. They allow developers to encapsulate logic that can be called multiple times throughout an application...";

echo "Date: " . formatDate($postDate) . "
";
echo "Slug: " . createSlug($postTitle) . "
";
echo "Preview: " . truncateText($postContent, 50) . "
";

/* Output:
Date: April 10, 2025
Slug: how-to-use-php-functions-effectively-in-your-projects
Preview: PHP functions are an essential part of creating...
*/
?>

Best Practices for PHP Functions

  • Give your functions meaningful, descriptive names that explain what they do.
  • Keep functions small and focused on a single task (single responsibility principle).
  • Document your functions with comments, especially for complex logic.
  • Always validate function parameters to ensure they meet expected criteria.
  • Use type hints (PHP 7+) to specify parameter and return types for better code quality.
  • Return values instead of echoing them directly, to make functions more reusable.
  • Avoid using global variables inside functions when possible; pass parameters instead.
  • Group related functions into classes or modules for better organization.
  • Test your functions to ensure they work correctly in all scenarios.
